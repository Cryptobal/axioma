## Por qué integrar
Sin integraciones, el ERP se convierte en un silo. Con APIs modernas, habilitas automatización, reportes confiables y experiencias digitales coherentes.

## Patrones
- API Gateway y autenticación (OAuth2, API Keys con rotación, scopes mínimos)
- Webhooks y colas para eventos (reintentos, idempotencia, firma de mensajes)
- ETL controlado para legados (ventanas, CDC, validaciones y trazabilidad)
- Versionado y compatibilidad (contratos, pruebas contractuales)

## Buenas prácticas
Trazabilidad end-to-end, observabilidad (métricas, logs, trazas), límites de tasa, pruebas contractuales y documentación viva (OpenAPI + ejemplos).

## Primeros pasos
Define un dominio acotado (ej. órdenes), diseña contratos y habilita un piloto con KPIs: tiempo de ciclo, errores y cobertura de eventos.

## Arquitectura de referencia
- Front: Next.js
- Backend: Python/Node
- Broker: colas (ex. SQS/Cloud Tasks)
- Persistencia: Postgres/Neon
- Cache (opc.): Redis

## FAQ
**¿Cómo asegurar webhooks?** Firma HMAC y reintentos exponenciales.

**¿Cómo migrar sin detener operaciones?** Corre ambas rutas (v1/v2) y depreca con fecha.
